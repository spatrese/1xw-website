# 1XW Trading – Research Platform

1XW Trading is a systematic macro research framework designed to identify and execute cross-asset trading opportunities across equities, commodities, FX and rates.

The platform produces weekly research outputs and a model portfolio which are published through a lightweight static website.

The project combines:

- Python-based data generation
- JSON research outputs
- A static web interface for visualization

---

## System Architecture

The system consists of three layers:

1. Data generation (Python scripts)
2. JSON outputs
3. Website visualization

```text
1XW System

Python research engine
        │
        ▼
   JSON outputs
        │
        ▼
  Static website
```

Python generates structured JSON files which are consumed directly by the website.

This architecture allows the site to update automatically whenever the research scripts are run.

## Website Pages

### Landing Page

`index.html`

The landing page introduces the framework and displays the public performance charts.

Sections include:

- System overview
- Framework principles
- Performance logic
- NAV chart
- PLB chart
- Transparency structure

### Weekly Research

`app.html`

This page displays the weekly research output generated by the system.

Sections include:

#### Technical Breadth
Cross-asset market state across the main asset classes.

#### Technical Overview
Instrument-level technical signals including:

- setup classification
- signal score
- 20-day return
- 60-day return

#### Fundamental Overview
Macro and thematic context affecting the instruments.

#### Top Ideas
The highest conviction trade setups generated by the model.

#### Macro Context
Qualitative commentary describing the macro environment.

#### Event Calendar
Key upcoming macro or earnings events relevant for markets.

Data source:

```text
content/site_weekly.json
```

### Model Portfolio

`model.html`

The model portfolio page displays performance and portfolio activity generated by the trading framework.

Sections include:

#### Snapshot

Displays:

- number of open positions
- current NAV
- NAV vs PLB gap

#### Performance

Chart of model portfolio NAV.

#### PLB

Chart of Performance Lower Boundary (risk control framework).

#### Model Trades

Table showing historical model trade signals.

Data sources:

```text
content/site_performance.json
content/site_screener.json
```

### Login Page

`login.html`

Simple front-end authentication used to restrict access to the research pages.

Authentication is implemented via browser storage:

```text
localStorage token
```

This system is sufficient for a private research site but is not intended as a secure authentication system.

## Weekly Production Workflow

The site is updated weekly using the following process.

### 1. Update trade blotter

Update the model blotter Excel file.

### 2. Generate screener data

```bash
python build_from_blotter_excel.py
```

### 3. Compile weekly research

```bash
python weekly_compiler.py
```

This generates the JSON files used by the website.

### 4. Commit changes

```bash
git add .
git commit -m "Weekly update"
git push
```

Once pushed, the website automatically displays the updated research.

## Automation

The weekly update process can be automated using a batch script.

Example:

```text
weekly_update.bat
```

Contents:

```bat
cd "1XW Website"

python build_from_blotter_excel.py
python weekly_compiler.py

git add .
git commit -m "Weekly research update"
git push
```

Running the batch file completes the full weekly update.
